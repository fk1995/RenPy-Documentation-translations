对话与叙述¶

文本是视觉小说的基础，也是所有基于剧情的游戏的重要组成部分。文本是由对话和叙述组成的。（方便起见，我们将会把对话和叙述统称为对话，除非在某个方面它们之间会有很大的区别。）同样地，用户能够自由地调整对话窗口的外观也是很重要的。

在Ren'Py中，大部分对话使用say命令来完成。对话的文本外观可以使用Character对象来为每名角色进行自定义。

Say命令¶

Say命令用于显示对话和叙述。因为它几乎总是在Ren'Py脚本中使用频率最高的一个命令，所以我们尽可能地简化了say命令的语法。以下是say命令的几个例子：

    "这是叙述。"

    "Eileen" "这是个对话，显式地提供了角色名称。"

    e "这是使用了一个Character对象的对话。"

    "乓！" with vpunch

say命令的第一种形式是一个字符串。这种形式用来显示叙述文本，而字符串的内容就是叙述的内容。

say命令的第二种形式是两个字符串。第一个字符串是正在说话的角色名称，第二个是说话的内容。

第三种形式是一个简单表达式，后面跟着一个字符串。简单表达式的值应该是一个字符串或者一个Character对象。如果是个Character对象的话，该对象将会被用于决定对话的外观以及其他设置。

最后一种形式是一个字符串加上一个with从句。在上面这个例子里，该字符串将会被显示，同时屏幕会震动起来。

尽管具体的细节是由该say命令使用的Character对象决定的，但通常情况下say命令被用于显示文本，并等待用户点击，然后从屏幕上消去该文本。

在Ren'Py里，某些字符是特别的，因此在使用的时候应该特别注意。{字符是一个文本标记的开始字符，[ 字符是一个文本替换的开始字符。若想在对话中使用它们，您必须双写它们。若想使用引号，您也必须先使用反斜杠来进行转移，以免它不正确地结束当前字符串。栗例子：

    "我听到了一个基佬的怒吼，\"让菊花盛开得更壮丽些吧！\""

定义Character对象¶

通过定义一个Character对象并在say命令中使用它，您可以自定义对话的外观（以及从某种程度上，对话的处理方式）。Character对象的创建方式是使用define命令将一个Character对象赋值给一个变量。如：

define db = Character("逗比",
                     who_color="#c8ffc8")

定义好之后，就可以在say命令里使用这个Character对象了：

    db "大家好，我是逗比。"

Character实际上是个python函数，而且有很多参数。这些参数将会控制该Character对象的行为。

define命令会对提供的表达式求值，然后将其赋值给指定的名称。如果不在init区块内的话，define命令会自动以优先级0的init区块运行。（也就是说，无论您是否在init区块中使用define，define命令一定会在init区块内运行，且优先度默认为0。）

Character(name, kind=adv, **args)¶

创建并返回一个Character对象，用于控制对话和叙述的外观和行为。

name
如果是个字符串，那此字符串就是该角色的姓名。当 name的值是None的时候，不会显示角色的名称，正如叙述的时候不会显示叙述者的名称一样。
kind
该Character对象的类型。如果提供此参数，所有其他未提供的参数值将采用kind指定的Character对象的参数。此参数可以用来创建一个模板Character对象，其他的Character对象可基于该模板对象，在其基础上稍作修改。

关联图像 一个Character对象可以与一个图像标记相关联。这将允许使用该角色的say命令显示一张与此标记相应的图像，也使得Ren'Py可以为该角色自动选择对话框头像。

image
指定与该Character对象关联的图像标记的字符串。

语音标记 若该Character对象拥有语音标记，则可以在设置界面内调整其语音音量。

voice_tag
该字符串将允许在设置界面调整该角色的语音音量。

前缀与后缀 使用这些参数，您可以在角色姓名和对话内容前后加上一些前缀和后缀。这可以用于为该角色的对话添加引号。

what_prefix
添加到对话内容之前的字符串。
what_suffix
添加到对话内容之后的字符串。
who_prefix
添加到角色姓名之前的字符串。
who_suffix
添加到角色姓名之后的字符串

改变显示的姓名 这些选项可以用来调整角色的姓名显示。

dynamic
若值为True，name应该是含python表达式的字符串。在该角色每次说话前，该字符串将被求值，求得的值将作为该角色的姓名。

交互控制 这些选项将会控制对话是否显示，交互是否发生，已经显示对话时Ren'Py进入的状态。

condition
如果提供该参数，它应该是含有一个python表达式的字符串。若该表达式值为False，则对话不会发生，就好像这条say命令不存在一样。
interact
如果值为True，也就是默认值，那么当对话被显示的时候将进入一个交互。若为False，则不会进入交互，在此时您可以向屏幕上添加其他的元素，然后手动进入交互状态。
mode
指定该角色说话时进入的系统状态的字符串。详情见系统状态这一节。
callback
当该角色说话时某事件发生后调用的函数，详情见 角色回调函数这一节。此参数的值应该是一个能够接受并处理事件的函数。

等待点击标志 等待点击标志将会在所有的文本显示完毕之后出现，提示用户点击屏幕以继续。

ctc
此参数应该是一个作为等待点击标志的可显示对象，除非您提供一下更加详细的设置。
ctc_pause
当对话被{p}或者{w}暂停时，使用该可显示对象作为等待点击标志。
ctc_timedpause
当对话被{p=}或者{w=}暂停时，使用该可显示对象作为等待点击标志。若为None，将会使用ctc_pause的值。如果您想使用ctc_pause而不想要ctc_timedpause的话，请使用Null()作为此参数的值。
ctc_position
该参数控制等待点击标志的位置，若为 "nestled"，该标志将会出现在文本最后一个字符的后面。若为"fixed", ，该标志将会出现在一个固定的位置，由位置样式属性决定。

视屏 对话的显示使用了一个视屏。这些参数允许您选择相应的视屏，并向其提供参数。

screen
用于显示对话的视屏的名称。

以show_开始的关键字参数会把它们的值传递给此屏幕对应的参数（去掉show_这个前缀以后）。例如，show_side_image的值将会成为该视屏中side_image 的值。

一些常用的show_变量已被整合到了默认的say视屏中，如：

show_side_image
当提供一个可显示对象作为参数时，将会在显示对话的同时显示该可显示对象。该对象的位置由它的位置属性控制。这个参数一般用于显示角色头像。
show_two_window
如果为True，那么姓名框和对话框将会是两个独立的窗口。

调整文本样式和窗口样式 以who_, what_, and window_开始的参数，去掉这些前缀之后，将会用于调整角色姓名，对话内容，以及包含两者的窗口的样式属性。

比如，如果一个Character对象有关键字参数 who_color="#c8ffc8"，它的姓名将会变成绿色。window_background="frame.png"这样的参数将会改变现实该角色对话窗口的背景。

角色姓名，对话内容和窗口的样式属性也能通过who_style, what_style, 和 window_style参数来设置。

带Image属性的say命令¶

当一个Character对象带有图像标签的时候，say命令的第一部分和第二部分之间可以插入图像属性。

在这种情况下，若当前该标签的图像正被显示在屏幕上，则Ren'Py会自动执行一条show命令，显示该图像标签以及相应属性对应的图像。若该标签的图像没有出现在屏幕上，Ren'Py会将图像属性用于显示对话框头像图片，但是并不会执行show命令。

作为例子，可以看看下面的代码：

define e = Character("Eileen", image="eileen")

label start:

    show eileen mad
    e "我对你有些失望。"

    e happy "不过现在都无所谓了。"

它等价于：

define e = Character("Eileen")

label start:

    show eileen mad
    e "我对你有些失望。"

    show eileen happy
    e "不过现在都无所谓了。"

若您希望当以此法改变图像时，有一个转场效果的话，您可以将 config.say_attribute_transition设定为您希望的转场效果来达到这一点。

Character对象范例¶

下面是一些Character对象的范例：

#此Character对象的对话都在一对半角括号内。
define skydark = Character("skydark", what_prefix='(', what_suffix=')')

#此Character对象的名称由一个变量(player_name)决定。
define p = Character("player_name", dynamic=True)

特殊的Character对象¶

有些Character对象是预定义的，会在特定场合下被自动使用。若您手动重新定义这些角色，则Ren'Py的某些行为将会被改变。但如果是无意中使用这些角色，则可能会造成一些意料之外的问题。

adv
Character对象默认的种类（即kind参数的默认值）。此Character对象只会在屏幕上显示一行对话。
nvl
在 小说模式中显示对话的Character对象，会在屏幕上显示多行对话。
narrator
用于显示叙述的Character对象，在没有角色姓名的say命令中使用。
name_only
在角色名称是字符串的say命令中使用的Character对象。Ren'Py会创建一个该对象的一个副本，将其名称设为接收到的字符串，然后使用新创建的Character对象来显示此对话。
centered
该Character对象的对话将显示在屏幕下方的正中央（与对话窗口的位置无关）。
vcentered
该Character对象的对话将显示在屏幕的正中央（与对话窗口的位置无关）。
extend

A character that causes the last character to speak to say 用于延续前一个角色的对话的Character。这样，您可以在对话的过程中改变屏幕的内容。

extend在小说模式下也会正常地工作。

例子：

#显示第一行对话，等待点击，改变表情，然后显示
#剩下的部分。

show eileen concerned
e "有时候，我心情不太好，比如大姨妈来了。"
show eileen happy
extend "但是我一般能很快恢复过来！"

#以下代码效果相似，唯一的不同在于第一行结束后没有等待点击。
#仅当玩家没有把文字速度设得很高的时候才有区别。 
show eileen concerned
e "有时候，我心情不太好，比如大姨妈来了。{nw}"
show eileen happy
extend "但是我一般能很快恢复过来！"

对话窗口管理¶

Ren'Py提供了几个命令，使您能够自由地控制对话窗口。在对话交互的过程中，对话窗口是不会被隐藏的，因此这些命令用于在非对话交互中控制对话窗口的出现和隐藏。

window show

window show命令会显示对话窗口。它可以带一个转场效果作为参数，如果没有提供此参数，config.window_show_transition 将被使用。

window hide

window hide命令会隐藏对话窗口。它可以带一个转场效果作为参数，如果没有提供此参数，config.window_hide_transition将被使用。

window auto

此命令将开启窗口自动管理。窗口将在config.window_auto_show中指定的命令前被显示（默认情况下是say命令）；在 config.window_auto_hide中指定的命令前被隐藏（默认情况下是scene命令）。

window auto命令使用config.window_show_transition 和config.window_hide_transition来显示和隐藏对话窗口。 window auto会在使用window show或window hide后无效。

如：

window show # 以默认转场效果显示对话窗口。
pause       # 对话窗口在此暂停过程中出现。
window hide # 隐藏对话窗口
pause       # 对话窗口在此暂停过程中消失。

window show dissolve # 以dissolve转场效果显示窗口。
pause                # the window is shown during this pause.
window hide dissolve # 以dissolve转场效果隐藏窗口。
pause                #对话窗口在此暂停过程中消失。


window auto

"对话窗口将自动在该行之前出现。"
pause                # 窗口一直在屏幕上。

scene bg washington  #窗口将在scene命令之前被隐藏。
with dissolve

对话窗口管理命令受到设置中的“显示空窗口”的设定值的影响，具体见 Preference()。若此设定被设置为关闭状态，这些命令将会无效。
